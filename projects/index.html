Ritual Master - In Development

Repository
  • Developed in Unity using C#
  • Ritual Master was originally developed as a desktop application at Global Game Jam. After exhibiting at PlayThrough Game Convention we found that users enjoyed the puzzles much more than the other features of the game. Listening to that feedback we decided to pivot and make a mobile game centered around those puzzles.
  • The puzzles consists of mastermind style game play growing in difficulty through size and number of resources. The star pattern is drawn dynamically based on the number of sockets. This allows us to scale the difficulty of various levels and modes without needing excess assets.
  • The goal of the user to find the correct pattern to complete the ritual

Particle Filter

  • Developed in C++ using the ArrayFire library, Prototyping done in Matlab
  • Developed a GPU-accelerated Particle Filter for use in tracking moving objects in video. This model is based on a template image of the object being tracked and can be used for many different applications. The goal was to maintain accuracy of tracking while reducing tracking times.
  • Parallelizing for GPU significantly accelerated the task over conventional serial methods. This is done by collecting the image patches for each particle in parallel and vectorizing the calculations that determine the probability that the object is located at that particle.
  • Further works needs to be done to optimize for multi-object tracking and improve the robustness of the template matching.

GoNN - In Development

Repository
  • GoNN is a minimalist Neural Network package for Go, a personal project that I am currently developing.
  • The plan is to develop a simple and easy to understand Neural Network package that will enable users to easily set up and test a network.
  • The source will not be highly optimized for speed, but easy to read through and understand what is happening at each step. There will be some sacrifice in training times, but it should be sufficient for many tasks.
  • I plan to use this package and Go as the backend for a web tool that visualizes the network and provides users with a means to create and see the results of various network structures.

GatherMate

Repository
  • Developed in Android Studio (Java) using Firebase and Google Maps API.
  • This is a prototype application for spontaneous event scheduling. Each user can create events that are visible to friends. Friends can be added via username or email and are shown any events that their friends create.
  • Each event opens a details page that contains a map view and other information about the event. Navigation to the event can be started from the map view and will open in Google Maps.
  • Timers countdown to indicate when an event starts. Colored travel times indicate whether you can make it to an event on time and are updated from current traffic conditions via the Maps API.
